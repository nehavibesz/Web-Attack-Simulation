# Web Attack Simulation

## Overview

This project demonstrates practical web application attack simulations performed in a controlled lab environment using DVWA (Damn Vulnerable Web Application). Network traffic generated during the attacks was analyzed using Wireshark, while Burp Suite was used to automate specific attack scenarios.

## Objectives

* Understand common web application vulnerabilities.
* Observe HTTP traffic generated during attacks.
* Analyze authentication mechanisms and response behavior.
* Gain hands-on experience with cybersecurity tools.

## Tools Used

* Kali Linux
* DVWA (Damn Vulnerable Web Application)
* Wireshark
* Burp Suite
* Firefox

## Lab Environment

The experiments were conducted locally inside a Kali Linux virtual machine in an isolated educational environment.

## Attacks Demonstrated

### SQL Injection

Examined vulnerable input fields and analyzed HTTP requests generated during SQL injection attempts.

### Brute Force Attack

Used Burp Suite Intruder to automate credential testing and identify successful authentication attempts through response analysis.

## Key Learning Outcomes

* Packet analysis using Wireshark.
* HTTP request and response inspection.
* Understanding SQL injection concepts.
* Authentication bypass techniques.
* Brute-force attack methodology.
* Importance of secure coding and HTTPS.

## Files Included

* Network Security Lab Report

## Disclaimer

This project was conducted strictly for educational purposes inside a controlled laboratory environment. No unauthorized systems were targeted.
